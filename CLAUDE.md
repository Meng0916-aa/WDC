# CLAUDE.md

本文件用于向 Claude Code 提供本项目的核心背景与工作约定。请在执行任何任务前先阅读并遵守。

## 1. 项目概述

本项目是 **CoCrNi 粉末在 316L 不锈钢基板上的稳恒磁场辅助多道激光熔覆温度场分析项目**。

研究目的是利用红外温度场数据，结合稳恒磁场对熔池行为的影响，分析熔覆过程中的温度演化与温度梯度特征，为后续 PyTorch 时序预测建模与 SCI 论文图表分析提供数据支撑。

### 1.1 实验参数

| 参数 | 数值 |
| --- | --- |
| 粉末材料 | CoCrNi |
| 基板材料 | 316L 不锈钢 |
| 激光功率 | 450 W |
| 扫描速度 | 800 mm/min |
| 送粉速率 | 40 g/min |
| 磁场强度 | 0、40、80、120 mT |
| 道间距 / 搭接距离 | 0.8 mm |
| 熔覆方式 | 稳恒磁场辅助多道激光熔覆 |

### 1.2 数据来源

- 红外相机：**Xiris VXIR-3000**
- 原始数据格式：`.xtherm`
- 当前阶段**尚无真实实验数据**，但所有代码必须按真实数据工作流来设计；测试时可以使用极小的临时二进制 fixture（tiny temporary binary fixture）来进行单元测试。

## 2. 工作流目标

实验完成后的整体工作流如下：

1. 将真实的 `.xtherm` 文件登记（注册）到本地 SQLite 数据库中（仅保存路径和元数据，**不存入原始文件内容**）。
2. 根据数据库记录自动读取对应的 `.xtherm` 文件。
3. 对 `.xtherm` 文件进行格式探测与解析，将其转换为温度矩阵（时间序列上的二维温度场）。
4. 计算每一帧的表面温度梯度：`Gx`、`Gy`，以及合成梯度 `G = sqrt(Gx^2 + Gy^2)`。
5. 提取关键特征：`Tmax`、`Tmean`、`Gmax`、`Gmean` 等。
6. 将特征结果回写数据库，供后续 PyTorch 时序预测与 SCI 论文图表分析使用。

> 注意：当前阶段**不要**编写复杂的深度学习模型；优先完成数据库、`.xtherm` 读取、格式探测、温度梯度计算和特征提取等基础能力。

## 3. 重要约定（必须严格遵守）

1. **当前还没有真实实验数据**，但代码必须按真实数据工作流设计。
2. **不要**以 `synthetic experiment pipeline`（合成实验流水线）作为主流程；合成数据只允许出现在单元测试中。
3. 可以使用 **tiny temporary binary fixture** 做单元测试，但不要把它当作生产数据源。
4. 原始 `.xtherm` 文件**不存入数据库**，数据库中只保存文件路径和元数据（如实验编号、磁场强度、扫描速度、采集时间、文件大小、帧数等）。
5. 本地数据库路径固定为：`database/thermal_cladding.db`（相对于项目根目录）。
6. 所有路径都应相对于项目根目录 **`D:\GEJ-WDC`** 来书写；代码内部统一使用 `pathlib.Path` 或正斜杠形式表示。
7. 代码优先使用以下技术栈：
   - Python（主语言）
   - SQLite（本地数据库）
   - NumPy、SciPy（数值计算、梯度计算）
   - Pandas（特征表、结果汇总）
   - PyYAML（配置文件）
   - pytest（单元测试）
8. 当前阶段**不要**写复杂的深度学习模型；先把数据库、`.xtherm` 读取、格式探测、温度梯度计算和特征提取等基础部分做扎实。
